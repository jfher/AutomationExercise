## 1. Purpose

This document defines the testing activities, scope, resources and execution plan for the Automation Exercise Enterprise QA Framework.

---

## 2. Scope

### In Scope

- User Registration
- User Authentication
- Product Catalog
- Product Search
- Shopping Cart
- Checkout
- Contact Form
- Account Deletion
- Public API

### Out of Scope

- Performance Testing
- Security Testing
- Mobile Native Testing
- Database Validation
- Third-party integrations

---

## 3. Test Objectives

The objectives of this project are:

- Validate critical business flows.
- Verify functional correctness.
- Detect regressions.
- Build a maintainable Playwright framework.
- Demonstrate enterprise QA practices.

## 8. Risks

The highest project risks are:

- Checkout failures
- Authentication failures
- Shopping Cart inconsistencies

Testing priority follows the Risk Analysis document.

---

## 9. Entry Criteria

Testing starts when:

- Environment is operational.
- Required data exists.
- Framework is available.

---

## 10. Exit Criteria

Testing finishes when:

- Smoke suite passes.
- Critical scenarios pass.
- No blocker defects remain.

---

## 11. Success Metrics

The project will measure:

- Test execution success rate.
- Automation coverage.
- Number of detected defects.
- Test execution duration.
- Flaky test rate.
